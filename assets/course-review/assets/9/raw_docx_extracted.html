OPCIÓN METODOLÓGICA DEL ESPACIO ACADÉMICO (redacción dirigida al estudiante)

El espacio académico Bases de Datos II cuenta con un énfasis teórico - práctico, por lo que la metodología del curso se centra en “Aprendizaje basado en proyectos”. Por esta razón a lo largo del curso encontraremos actividades como resolución de problemas, resolución de casos, talleres de práctica y un proyecto integrador.

PRESENTACIÓN DEL ESPACIO ACADÉMICO (No superar 1 página) (redacción dirigida al estudiante)

Bienvenidos al espacio académico Bases de Datos II. El sentido de este espacio es permitir el desarrollo de la capacidad para determinar la configuración, administración y optimización de sistemas gestores de bases de datos documentales.

Para lograr esta meta de aprendizaje, el curso se encuentra organizado en 3 unidades de estudio. La primera unidad busca mostrar una introducción a las bases de datos no estructuradas, así como el modelado y diseño de las estas. La segunda unidad busca presentar el proceso de creación de bases de datos documentales haciendo uso de un motor o sistema de gestión de base de datos; adicionalmente, dar a conocer las principales consultas (querys) que podemos realizar sobre estas herramientas de almacenamiento. Y, finalmente, la tercera unidad busca conocer cómo relacionar las diferentes colecciones mediante índices; como complemento, se aplicarán estos conocimientos en la integración de un motor de bases de datos con un lenguaje de programación.

Con estos elementos el curso te aportará al desarrollo de la competencia del programa:

  • Diseñar bases de datos documentales, identificando y estructurando eficazmente los datos no estructurados.
  • Administrar bases de datos documentales, incluyendo la inserción, actualización y eliminación de documentos, así como la creación de consultas eficientes para recuperar información relevante.
  • Aplicar conocimientos de bases de datos documentales en contextos prácticos, como la integración de estas bases de datos en aplicaciones web y móviles, la implementación de sistemas de gestión de contenido, y la resolución de problemas del mundo real en la industria.

ACTIVIDAD 8: Entrega del proyecto final

¿Qué vamos a lograr?

Aplicar los conocimientos vistos durante el curso mediante la creación de una aplicación que haga uso de una base de datos documental.

¿Cómo lo vamos a lograr?

Apreciado estudiante,

A través de esta actividad de manera grupal vamos a desarrollar una aplicación, haciendo uso de su lenguaje de programación de preferencia, que permita resolver el siguiente caso de estudio

Descripción:

Desarrolle una aplicación de tienda que permita a los usuarios realizar las siguientes operaciones:

  • Ver Todos los Productos: Mostrar todos los productos disponibles en la tienda.
  • Buscar Productos por Categoría: Permitir a los usuarios buscar productos por su categoría.
  • Agregar un Producto a la compra: Los usuarios deben poder agregar productos a su carrito de compras.
  • Ver el Contenido del Carrito: Mostrar todos los productos que el usuario ha agregado al carrito.
  • Realizar una Compra: Simular el proceso de compra donde los productos en el carrito se marcan como vendidos y se eliminan del inventario.

Nota: la aplicación puede ir local (app de escritorio) no es necesario hacer web. La interfaz puede ir por consola

Requisitos Técnicos:

  • Utilice el lenguaje de programación de su preferencia.
  • Utilice una biblioteca para conectarte a la base de datos MongoDB.
  • Diseñe al menos dos colecciones MongoDB: una para los productos y otra para los usuarios.
  • Estructura de la Base de Datos: esta estructura está definida de acuerdo con el análisis que realicen sobre el ejercicio propuesto, cada grupo debe proponer las colecciones y atributos que consideren necesarios.
  • Interfaz de Usuario: Crea una interfaz de usuario para tener interacción con la tienda.
  • Operaciones CRUD: Utiliza las funciones de la biblioteca pymongo u otra para realizar operaciones CRUD (Crear, Leer, Actualizar, Eliminar) en las colecciones de MongoDB propuestas.

Puntos Adicionales (Opcional):

  • Autenticación de Usuarios: Implemente un sistema de autenticación de usuarios para que cada usuario tenga su propio carrito e historial de compras.
  • Gestión de Inventario: Implemente lógica para controlar el inventario de productos, asegurando que los productos no se puedan vender si no hay suficientes en stock.

Como producto de la actividad entrega a través de este medio entregar:
