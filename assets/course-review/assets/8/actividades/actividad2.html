ACTIVIDAD 2: Caso de estudio sobre bases de datos documentales

¿Qué vamos a lograr?

  • Desarrollar a través de un caso de estudio, un diseño de una base de datos documental en el que se evidencie la aplicabilidad de los modelos conceptuales y lógicos vistos.

¿Cómo lo vamos a lograr?

Apreciado estudiante,

A través de esta actividad de manera grupal vamos a diseñar una base de datos documental a partir del siguiente caso de estudio:

Descripción del Caso:

Imagina que estás trabajando en el diseño de una plataforma de redes sociales similar a Twitter o Instagram. Esta plataforma debe ser capaz de manejar grandes volúmenes de datos en tiempo real, como publicaciones de usuarios, comentarios, me gusta y seguimiento de usuarios. Dado que la escalabilidad y la velocidad son cruciales, se decide utilizar una base de datos NoSQL para este proyecto.

Requisitos Iniciales:

  • Almacenar publicaciones de usuarios, incluyendo texto, imágenes y videos.
  • Registrar información sobre los usuarios, como nombres, alias y detalles de perfil.
  • Gestionar seguidores y seguidos por cada usuario.
  • Registrar la actividad de usuarios, como me gusta, comentarios y compartidos.
  • Permitir búsquedas rápidas y filtrado de publicaciones por diversos criterios.
  • Mostrar el feed de noticias personalizado de cada usuario basado en a quién siguen.

Para iniciar revisa atentamente los siguientes materiales:

  • Cómo modelar NoSQL: https://www.youtube.com/watch?v=Zdlude8l8w4
  • Modelamiento de datos: https://www.mongodb.com/docs/manual/core/data-modeling-introduction/
  • Herramienta de modelamiento de datos: https://jsoncrack.com/

Luego de haber revisado los materiales y el caso propuesto, entregar:

  1. Documento en .pdf en el que se evidencie el diseño conceptual y lógico de la base de datos documental según el caso de estudio propuesto.
